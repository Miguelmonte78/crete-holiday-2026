Information used by the website

What may be stored locally

Visited places and activities: when a place, beach, restaurant or activity is marked as visited, the website stores that status locally in the browser.
Packing checklist progress: checked packing items may be saved locally so the checklist remains useful after refreshing the page.
Actual expense entries: spending tracker entries such as amount, category, day and description may be saved locally for budget comparison.
Restaurant meal cost entries: restaurant visit status and actual meal costs may be saved locally and used by the budget dashboard.
Receipt OCR text: if receipt scanning is used, OCR may process the image in the browser and prefill fields for manual confirmation. Do not rely on OCR without checking the values.
Theme preference: dark mode or light mode preference may be stored locally.

Where data is stored

LocalStorage and browser data

Local-first design

The website is designed as a static planner. Trip interaction data is intended to stay on the device and browser where the website is used.

Local browser storage No login No backend database

Device-specific information

Information saved on one phone or computer may not automatically appear on another device, because the website uses browser storage rather than a shared account.

Per browser Per device Not synchronized automatically

Third-party services

External tools and links used

Bootstrap CDN: used to load visual layout components when internet access is available.
Google Fonts: used for website typography.
Google Maps links: direction buttons open Google Maps in a new tab/app. Google Maps is outside this website.
OpenStreetMap / Leaflet: the maps page may load map tiles from external map providers.
Chart.js / OCR libraries if used: budget charts and receipt scanning may use external JavaScript libraries depending on the page version.
Tripadvisor or restaurant links: external restaurant/review/photo pages may have their own privacy/cookie policies.

Receipt scanning

Receipt OCR privacy notes

Manual confirmation

OCR is used only to help prefill fields. The family should check restaurant name, date, amount and category before saving.

Avoid sensitive receipts

Do not upload or scan documents containing unnecessary personal, banking, medical or passport information.

Delete if needed

If an incorrect or sensitive expense is added, delete the entry or clear browser data for the website.

Deleting data

How to remove locally saved data

  1. Open the browser settings on the phone or computer.
  2. Search for site data, cookies or local storage settings.
  3. Find the website domain or local file entry used for the Crete planner.
  4. Clear site data for this website.
  5. Refresh the website. Locally saved visited places, packing ticks and expenses should be reset.

Good practice

Recommended privacy rules for this trip site

Do not store passports
Keep passport copies in a secure phone folder, not inside this HTML website.
Keep insurance separate
Use the contact page as a note reference, not a secure document vault.
Check expense entries
OCR can be wrong. Always confirm before saving actual costs.
Use trusted devices
Avoid entering trip information on public/shared computers.

Contact

Questions or corrections

For this personal travel website, privacy questions or corrections should be handled by the family member maintaining the website files.
